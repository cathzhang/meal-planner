# 今天吃什么 — 项目需求文档

---

## 一、项目定位

**家庭菜单推荐系统**

根据家庭成员、季节、时间、预算等约束，智能推荐"今天吃什么"，并生成购物清单。

---

## 二、菜品数据结构

每道菜包含以下字段：

| 字段 | 说明 |
|---|---|
| **菜名** | — |
| **食材** | 主要食材（普通油盐等基础调料不列入） |
| **特殊酱料** | 需要单独采购的调味料（豆瓣酱、蚝油等） |
| **制作时间** | 灶上实际制作时间（分钟） |
| **备菜时间** | 切配、腌制等耗时（分钟） |
| **难易程度** | 简单 / 中等 / 困难 |
| **菜品类型** | 大荤 / 小荤 / 全素 |
| **夏日推荐** | 是 / 否 |
| **冬日推荐** | 是 / 否 |
| **辣味等级** | 不辣 / 可免辣 / 辣 |
| **儿童友好** | 是 / 否 |
| **适合月份** | `[1-12]`，空数组表示全年都有 |
| **菜价评估** | 便宜 / 中等 / 较贵 |
| **预制菜** | 是否有成熟预制菜可选 |
| **变体组** | 如"排骨汤系列"，同组菜可互相替换 |
| **喜好人员** | 家庭成员名单 |
| **汤菜** | 是 / 否 |

### 字段演进记录

- **时间拆分**：从单一"制作时间"拆分为"制作时间 + 备菜时间"
- **辣味合并**：从"是否辣 + 是否可去辣"两个布尔值，合并为"不辣 / 可免辣 / 辣"三选一
- **季节细化**：从夏/冬布尔值，改为月份列表（支持跨季节食材）

## 三、用餐场景配置

推荐前需确定的参数：

| 参数 | 说明 |
|---|---|
| **吃饭人数** | 决定做几道菜 |
| **工作日 / 周末** | 工作日只推快手菜（≤35分钟），周末可推费工夫大菜 |
| **是否需要汤** | 默认配 1 个汤 |
| **主菜数量** | 默认 `n人 → n-1 个主菜`，可手动覆盖 |
| **已有菜品** | 剩菜 / 打包菜，会从推荐中扣除 |

### 配餐公式示例

| 人数 | 默认配置 | 说明 |
|---|---|---|
| 2 人 | 1 主菜 + 1 汤 | 或 2 主菜（不选汤） |
| 4 人 | 3 主菜 + 1 汤 | 如有剩菜则相应减少 |

## 四、数据存储方案

```
data/recipes.json（文本文件，Git 追踪）
        ↓
  python import_data.py
        ↓
meal_planner.db（SQLite 数据库，运行时生成）
```

- **JSON 是核心数据**：人类可读，Git  diff 清晰，丢失可随时恢复
- **数据库是运行时副本**：程序查询用，不加入 Git

### 配套工具

| 工具 | 功能 |
|---|---|
| `add_recipe.py` | 交互式一问一答录入新菜品 |
| `cli.py` | 命令行增删改查（list / show / search / add / edit / delete / stats） |
| `import_data.py` | 从 JSON 重建数据库 |

## 五、推荐算法设计

### 为什么不适用传统推荐算法

- 没有用户评分矩阵 → 协同过滤 / 矩阵分解不可用
- 推荐目标是"一餐的组合"（2-4 道菜搭配），不是单个商品
- 约束条件多且硬：荤素、季节、时间、儿童、辣味、预算

### 推荐方案（三层架构）

#### 第一层：多因素加权评分

给每道菜按规则打分：

```
总分 = 季节匹配(+20) + 家人爱吃(+30) + 近期没做过(+50)
     + 预算合适(+10) + 时间充裕(+15) + 儿童友好(+25)
```

**特点**：逻辑透明、调参简单、冷启动友好

#### 第二层：组合优化

在一餐框架内找最优搭配：

```python
for 菜A in 大荤:
    for 菜B in 小荤:
        for 菜C in 全素:
            score = 组合评分(菜A, 菜B, 菜C)
```

组合评分额外考虑：
- 口味不重复（三道都是红烧味不太好）
- 制作时间可并行（两道炒菜同时进行）
- 总预算不超标

**特点**：直接解决"今天晚饭吃什么"这个组合问题

#### 第三层：基于内容的相似推荐

用食材重叠度（Jaccard 相似度）计算菜品相似性：

- "做了红烧排骨，配个什么素菜好？" → 推荐口味互补的素菜
- "家里没有冬瓜" → 推荐同组变体"玉米排骨汤"

#### 第四层：关联规则挖掘（预留）

积累一年菜单历史后，用 Apriori 算法挖掘"常一起做的菜"。

**示例规则**：
```
{清蒸鲈鱼} → {蒜蓉西兰花}  （支持度 80%）
```
